# 贡献指南
首先，感谢你对小可（与衍生项目）的关注。

我们欢迎任何形式的贡献，有关不同的帮助方式和处理方式的详细信息，请参阅[目录](#目录)。在做出贡献之前，请确保阅读相关部分。这将大大减轻我们维护者的负担，并为所有参与者带来更流畅的体验。
> [!NOTE]
> 另外，如果你喜欢本项目，但暂时不打算贡献也没关系。还有其他方法可以支持这个项目并表达支持：
> - 为本项目点 Star
> - 在你的项目 README 文件中引用本项目
> - 将机器人加入你的群聊/服务器
> - 将本项目/机器人分享或推荐给其他人

## 提出问题
> [!NOTE]
> 建议先阅读[《提问的智慧》](https://github.com/ryanhanwu/How-To-Ask-Questions-The-Smart-Way/blob/main/README-zh_CN.md)以避免带来不必要的麻烦。

在开始之前，请先搜索一下是否有相关的 [Issue](https://github.com/Teahouse-Studios/akari-bot/issues)。此外，一些显而易见的问题建议先在网络上搜索答案。

如果你仍有疑问需要联系开发者解答，我们支持以下方法：

- 在 Issue 中[提交新问题](https://github.com/Teahouse-Studios/akari-bot/issues/new)
- 加入 QQ 平台的[小可公共实例测试群（738829671）](https://qm.qq.com/q/Rmuo5ORYgq)或[茶馆工作室聊天群（979982065）](https://qm.qq.com/q/sjwFNX1NVC)

不管使用什么方法，请可能详细地描述你遇到的问题。

## 做出贡献

### 报告错误
> [!WARNING]
> **绝对不应该**在 Issue 或其他任何公开渠道报告或复现包含敏感信息的安全问题、漏洞或错误。
> 
> 敏感问题必须通过电子邮件（详见 [`pyproject.toml`](/pyproject.toml)）或私聊联系开发者解决。
>
> 违反此条规则的报告或消息将会被删除，执意违反则后果自负。

在开始之前，请先搜索一下是否有相关的 [Issue](https://github.com/Teahouse-Studios/akari-bot/issues)。如果没有类似的问题，请使用[模板](https://github.com/Teahouse-Studios/akari-bot/issues/new?template=report_bug.yaml)创建一个新 Issue。

请提供以下信息： 
- 错误的简要描述
- 复现问题的方法
- 预期结果和实际结果
- 错误信息（若有）

一份好的错误报告不应该让其他人需要追问更多信息。请尽可能详细描述问题，以避免不必要的困扰。

### 提出建议
在开始之前，请先搜索一下是否有相关的 [Issue](https://github.com/Teahouse-Studios/akari-bot/issues)。如果没有类似的问题，请使用[模板](https://github.com/Teahouse-Studios/akari-bot/issues/new?template=feature_request.yaml)创建一个新 Issue。

请提供以下信息： 
- 建议的简要描述
- 实现建议的方法
- 解释建议对项目更有用的原因（推荐）

若你已经写好对应的功能改进，在提交 PR 前建议在 Issue 提交建议，这样更方便我们进行讨论。

### 本地化
本项目的多语言部分目前由 Crowdin 托管，并由机器人定期维护本地化文件。你可以前往[这里](https://crowdin.com/project/akari-bot)参与改进小可的本地化支持。

为了避免不必要的报告堆积，除简体中文以外，不建议在 Issue 中提交关于本地化翻译的问题。

若你没有 Crowdin 账号，可联系开发者代为提交你的翻译文本。
